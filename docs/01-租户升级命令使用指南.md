# 租户升级命令使用指南

## 命令概览

```bash
php artisan app:tenancy-upgrade-command [选项]
```

多租户系统升级命令，采用三阶段执行模型，自动完成中央数据库变更、租户数据库变更和全局操作。

## 三阶段执行流程

每个版本按以下顺序执行：

```
1. centralUp()  → 中央数据库变更（执行一次）
2. tenantUp()   → 租户数据库变更（每个租户执行一次）
3. globalUp()   → 全局操作（执行一次，如菜单同步、系统参数更新）
4. 更新版本号    → 自动更新 admin_parameters.his_version
```

### 阶段说明

| 阶段 | 执行次数 | 数据库上下文 | 失败策略 |
|------|---------|-------------|---------|
| centralUp | 1 次 | 中央数据库 | 失败则停止整个升级 |
| tenantUp | 每个租户 1 次 | 租户数据库 | 失败记录错误，继续下一个租户 |
| globalUp | 1 次 | 中央数据库 | 失败记录错误，不阻止版本标记 |

## 命令选项

### 标准升级（无参数）

```bash
php artisan app:tenancy-upgrade-command
```

自动检测当前中央版本号，执行所有待升级版本。自动跳过已升级的租户。

### `--dry-run` 预览模式

```bash
php artisan app:tenancy-upgrade-command --dry-run
```

仅预览待执行的版本列表，不实际执行任何操作。会显示需要补跑的失败记录。

### `--ver` 指定版本

```bash
php artisan app:tenancy-upgrade-command --ver=1.0.3
```

只执行指定版本的升级，不受当前版本号限制。

### `--tenant` 指定租户

```bash
php artisan app:tenancy-upgrade-command --tenant=abc
```

只对指定租户执行 `tenantUp`，**跳过 `centralUp` 和 `globalUp`**。适用于单个租户升级失败后的补跑。

### `--force` 强制重跑

```bash
php artisan app:tenancy-upgrade-command --force --ver=1.0.3
```

强制重新执行指定版本，即使该版本已完成。**必须配合 `--ver` 使用**，不允许无条件全量重跑。

### 组合使用

```bash
# 强制对指定租户重跑指定版本
php artisan app:tenancy-upgrade-command --ver=1.0.3 --tenant=abc --force

# 预览指定版本的执行计划
php artisan app:tenancy-upgrade-command --ver=1.0.3 --dry-run
```

## 常见场景

### 场景 1：常规版本升级

```bash
# 1. 先预览
php artisan app:tenancy-upgrade-command --dry-run

# 2. 确认后执行
php artisan app:tenancy-upgrade-command
```

### 场景 2：某个租户升级失败

命令会自动跳过失败租户，继续升级其他租户。升级完成后查看摘要中的失败数。

```bash
# 修复问题后，只对失败的租户补跑
php artisan app:tenancy-upgrade-command --tenant=failed_tenant_id --ver=1.0.3
```

### 场景 3：自动失败补跑

命令启动时会自动检查 `upgrade_logs` 表中的失败记录，将需要补跑的版本和阶段纳入执行计划。无需手动指定，直接运行即可：

```bash
php artisan app:tenancy-upgrade-command
```

### 场景 4：需要完全重跑某个版本

```bash
php artisan app:tenancy-upgrade-command --force --ver=1.0.3
```

## 升级日志

所有升级操作都记录在中央数据库的 `upgrade_logs` 表中：

| 字段 | 说明 |
|------|------|
| version | 版本号 |
| phase | 执行阶段：central / tenant / global |
| tenant_id | 租户 ID（仅 tenant 阶段） |
| tenant_name | 租户名称 |
| status | 状态：running / success / error |
| started_at | 开始时间 |
| finished_at | 结束时间 |
| duration_ms | 耗时（毫秒） |
| error_message | 错误信息 |
| error_trace | 错误堆栈 |

## 版本文件结构

版本文件位于 `app/Upgrades/Versions/`，继承 `BaseVersion` 基类：

```php
class Version103 extends BaseVersion
{
    public function version(): string
    {
        return '1.0.3';
    }

    // 中央数据库变更（执行一次）
    public function centralUp(): void
    {
        $this->createTableIfNotExists('upgrade_logs', function (Blueprint $table) {
            // ...
        });
    }

    // 租户数据库变更（每个租户执行一次）
    public function tenantUp(): void
    {
        $this->tenantInfo('创建表 workflows');
        $this->createTableIfNotExists('workflows', function (Blueprint $table) {
            // ...
        });
    }

    // 全局操作（所有租户升级后执行一次）
    public function globalUp(): void
    {
        Artisan::call('db:seed', ['--class' => 'MenusTableSeeder', '--force' => true]);
        Artisan::call('app:update-menu-command');
    }
}
```

### 基类提供的辅助方法

| 方法 | 用途 |
|------|------|
| `$this->createTableIfNotExists($table, $callback)` | 表不存在时创建（幂等） |
| `$this->addColumnIfNotExists($table, $column, $callback)` | 列不存在时添加（幂等） |
| `$this->dropColumnIfExists($table, $column)` | 列存在时删除（幂等） |
| `$this->dropTableIfExists($table)` | 删除表（幂等） |
| `$this->runSeeder($class)` | 执行 Seeder |
| `$this->info($message)` | 输出日志 |
| `$this->tenantInfo($message)` | 输出带租户前缀的日志 |

## 自动生成版本文件

```bash
# 根据 git diff 自动生成升级脚本
php artisan app:generate-upgrade

# 指定基准 tag
php artisan app:generate-upgrade --tag=v1.0.2

# 指定目标版本号
php artisan app:generate-upgrade --ver=1.0.4

# 预览生成内容
php artisan app:generate-upgrade --dry-run
```

生成器会自动将 `admin/` 目录下的迁移放入 `centralUp()`，`tenant/` 目录下的迁移放入 `tenantUp()`，并使用幂等辅助方法。
